# Support Procedure
## password_reset
1. Open Identity Portal
2. Select Forgot Password
3. Verify identity
4. Set new password
5. Sign in again on all devices

## account_unlock
1. Open the corporate Identity Portal.
2. Choose Account Recovery or Unlock Account.
3. Verify the user’s identity through MFA, security questions, or corporate email.
4. Confirm the account was locked by failed login attempts.
5. Reset the account lock and notify the user to sign in.
6. Advise the user to use their corporate password manager and update saved credentials.

## mfa_registration
1. Open Identity Portal and sign in with corporate credentials.
2. Navigate to Security Settings or Multi-Factor Authentication.
3. Select Register a new authenticator device.
4. Scan the QR code with the approved authenticator app.
5. Enter the one-time code to verify the device.
6. Confirm that the new MFA method is active and remove any old or lost devices.

## vpn_connection_issue
1. Confirm the user is on a trusted network and has internet connectivity.
2. Verify the VPN client is installed and up to date.
3. Ensure device compliance checks have passed (endpoint protection, encryption, firewall).
4. Confirm the user has an active VPN policy and correct credentials.
5. Restart the VPN client and attempt reconnect.
6. If the VPN still fails, collect logs and escalate to Network Operations.

## email_access_issue
1. Confirm the user can sign in to the corporate identity account.
2. Verify mailbox provisioning status in the email administration portal.
3. Check for account issues such as suspended or blocked status.
4. Ensure the user is connecting with the correct email client settings.
5. If necessary, reset the email password and reconfigure the mailbox.
6. Advise the user to clear cached credentials and restart the email client.

## device_compliance_failure
1. Identify the compliance check that failed (OS update, endpoint protection, encryption, firewall).
2. Guide the user to apply the required updates or restart their device.
3. Verify the company’s device management agent is running.
4. Confirm disk encryption and firewall settings are enabled.
5. Re-run the compliance validation.
6. If the device still fails, escalate to IT Operations for remediation.

## new_employee_access_onboarding
1. Confirm the new hire’s employment agreement and manager approval are complete.
2. Provision the corporate identity account and assign the correct role-based access.
3. Enroll the device in endpoint management and security compliance.
4. Configure email, collaboration, and VPN access.
5. Ensure MFA registration is completed before first login.
6. Verify access to required onboarding tools and notify the manager when ready.
