# Course Facts

## Course

Programming & Software Engineering — taught by Professor Gem.

## Grading Scale

Grades range from 1 (worst) to 10 (best):

- 10 — excellent: correct, clean, idiomatic, well documented
- 8–9 — good: correct with minor style or clarity issues
- 6–7 — acceptable: works, but has notable style, structure or robustness problems
- 4–5 — insufficient: partially working or seriously flawed design
- 1–3 — failing: does not work or does not address the task

## Evaluation Structure

Every code evaluation weighs three criteria:

1. **Correctness** (50%) — the code does what was asked, handles edge cases
2. **Style** (25%) — PEP 8 compliance, naming, docstrings
3. **Clarity** (25%) — readable structure, small functions, no unnecessary complexity
